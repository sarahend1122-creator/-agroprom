## Агро-маркетплейс (MVP)

Международная платформа для продажи и аренды сельскохозяйственных активов (техника, земля и т.д.).

Этот репозиторий содержит **MVP-версию**: один fullstack-проект на базе **Next.js (React + TypeScript)** с простой архитектурой, удобной для дальнейшего расширения.

### Текущий стек

- **Frontend / Backend**: Next.js (App Router) + React + TypeScript
- **База данных**: PostgreSQL
- **ORM**: Prisma
- **Стили**: Tailwind CSS

### Быстрый старт

1. Установите Node.js LTS (рекомендуется последняя LTS-версия).
2. Установите зависимости:

```bash
npm install
```

3. Настройте переменные окружения:

Создайте файл `.env` в корне проекта (рядом с этим `README.md`) и добавьте строку подключения к PostgreSQL:

```bash
DATABASE_URL="postgresql://user:password@localhost:5432/agro_marketplace?schema=public"
```

4. Примените миграции Prisma (после создания схемы):

```bash
npx prisma migrate dev --name init
```

5. Запуск dev-сервера:

```bash
npm run dev
```

Приложение по умолчанию будет доступно по адресу `http://localhost:3000`.

### Структура проекта (план)

- `package.json` — зависимости и скрипты
- `app/` — страницы и маршруты Next.js (App Router)
- `app/(public)/` — публичные страницы (главная, список объявлений, детали)
- `app/(dashboard)/` — личный кабинет пользователя
- `app/api/` — API-роуты (авторизация, объявления, чаты и т.д.)
- `prisma/schema.prisma` — описание модели данных

### Ближайшие шаги по разработке

1. Добавить базовую структуру Next.js-проекта (App Router).
2. Описать Prisma-схему для пользователей, объявлений и чатов.
3. Реализовать базовые API-методы:
   - регистрация / авторизация;
   - создание и просмотр объявлений;
   - список объявлений с фильтрацией по стране/категории.
4. Реализовать простые экраны на фронтенде для MVP:
   - главная страница с поиском и фильтрами;
   - страница объявлений;
   - личный кабинет с управлением своими объявлениями.
