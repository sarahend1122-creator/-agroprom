# Деплой Agroprom на Vercel

## Ошибка: «Couldn't find any `pages` or `app` directory»

Она появляется, когда Vercel собирает проект **не из той папки**, где лежат `app/` и `package.json`.

### Что сделать (обязательно)

1. Открой **GitHub** → свой репозиторий с проектом.
2. Посмотри **корень репозитория** (самый верхний уровень):
   - Если видишь **папку** (например `rr`, `agroprom` или другое имя), а внутри неё — `app`, `package.json`, `next.config.mjs`, значит проект лежит в подпапке.
3. Зайди в **Vercel** → свой проект → вкладка **Settings**.
4. В блоке **General** найди **Root Directory**.
5. Нажми **Edit** рядом с Root Directory и укажи **имя этой папки** (например `rr` или как у тебя называется папка с `app` и `package.json`). Без слэшей в начале/конце.
6. Нажми **Save**.
7. Перейди на вкладку **Deployments** → у последнего деплоя нажми **⋯** → **Redeploy**.

После этого сборка должна идти из правильной папки и ошибка пропадёт.

---

### Если в корне репозитория сразу видны `app` и `package.json`

Тогда Root Directory должен быть **пустым** (или `/`). Если ошибка всё равно есть — проверь, что папка на GitHub называется именно `app` (с маленькой буквы).
